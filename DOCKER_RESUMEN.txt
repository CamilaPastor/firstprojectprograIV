  SERVICIOS:

  1️⃣  MYSQL (Base de Datos)
  ├─ Imagen: mysql:8.0
  ├─ Nombre: bolsa_empleo_mysql
  ├─ Puerto: 3306:3306
  ├─ Credenciales:
  │  ├─ Usuario: bolsa_user
  │  ├─ Contraseña: bolsa1234
  │  ├─ Base de datos: bolsa_empleo
  │  └─ Root password: root_password
  ├─ Volumen: mysql_data (persistencia)
  ├─ Script Init: BolsaEmpleo.sql
  │  ├─ Tabla: usuarios
  │  ├─ Tabla: puestos
  │  └─ Tabla: aplicaciones
  ├─ Health Check: mysqladmin ping
  └─ Red: bolsa_network

  2️⃣  APP (Aplicación Spring Boot)
  ├─ Build: Dockerfile (multi-stage)
  ├─ Nombre: bolsa_empleo_app
  ├─ Puerto: 8080:8080
  ├─ Variables de Entorno:
  │  ├─ SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/bolsa_empleo
  │  ├─ SPRING_DATASOURCE_USERNAME: bolsa_user
  │  ├─ SPRING_DATASOURCE_PASSWORD: bolsa1234
  │  ├─ SPRING_PROFILES_ACTIVE: docker
  │  └─ JAVA_OPTS: -Xmx512m -Xms256m
  ├─ Depends On: mysql (condition: service_healthy)
  ├─ Health Check: wget --spider http://localhost:8080/
  └─ Red: bolsa_network

  RED:
  ├─ Nombre: bolsa_network
  ├─ Driver: bridge
  └─ Subnet: 172.20.0.0/16

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🚀 COMANDOS PARA INICIAR                                                    │
└──────────────────────────────────────────────────────────────────────────────┘

  OPCIÓN A: Script Automático (RECOMENDADO)

    Windows:
    $ docker-init.bat

    Linux/Mac:
    $ chmod +x docker-init.sh
    $ ./docker-init.sh

  OPCIÓN B: Comandos Manuales

    1. Construir imagen:
       $ docker-compose build

    2. Iniciar servicios:
       $ docker-compose up -d

    3. Verificar estado:
       $ docker-compose ps

    4. Ver logs:
       $ docker-compose logs -f

    5. Acceder a aplicación:
       http://localhost:8080

┌──────────────────────────────────────────────────────────────────────────────┐
│ 📊 CICLO DE VIDA                                                             │
└──────────────────────────────────────────────────────────────────────────────┘

  STARTUP:
  1. docker-compose up -d
     ↓
  2. MySQL inicia
     ↓
  3. BolsaEmpleo.sql se ejecuta (inicializa BD)
     ↓
  4. Health check: mysqladmin ping ✓ healthy
     ↓
  5. App inicia (after MySQL is healthy)
     ↓
  6. Health check: GET / ✓ healthy
     ↓
  7. Aplicación lista para usar ✓

  SHUTDOWN:
  1. docker-compose down
     ↓
  2. App se detiene
     ↓
  3. MySQL se detiene
     ↓
  4. Red se elimina
     ↓
  5. Contenedores se eliminan
     ↓
  6. Volumen persiste (para siguiente startup)

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🗄️  BASE DE DATOS (BolsaEmpleo.sql)                                          │
└──────────────────────────────────────────────────────────────────────────────┘

  Tablas Creadas:

  1. usuarios
     ├─ id (PK)
     ├─ nombre VARCHAR(100)
     ├─ email VARCHAR(100) UNIQUE
     ├─ password VARCHAR(255)
     ├─ cedula VARCHAR(20) UNIQUE
     ├─ tipo_usuario (OFERENTE, EMPRESA, ADMINISTRADOR)
     ├─ descripcion TEXT
     ├─ activo BOOLEAN
     ├─ fecha_creacion TIMESTAMP
     └─ fecha_actualizacion TIMESTAMP

  2. puestos
     ├─ id (PK)
     ├─ titulo VARCHAR(150)
     ├─ descripcion TEXT
     ├─ empresa_id (FK → usuarios)
     ├─ salario_minimo DECIMAL
     ├─ salario_maximo DECIMAL
     ├─ ubicacion VARCHAR(100)
     ├─ tipo_contrato VARCHAR(50)
     └─ estado VARCHAR(50)

  3. aplicaciones
     ├─ id (PK)
     ├─ puesto_id (FK → puestos)
     ├─ oferente_id (FK → usuarios)
     ├─ estado VARCHAR(50)
     ├─ fecha_aplicacion TIMESTAMP
     └─ fecha_actualizacion TIMESTAMP

  Datos de Prueba:
  ✓ 3 usuarios (oferente, empresa, admin)
  ✓ Contraseñas hasheadas con BCrypt

┌──────────────────────────────────────────────────────────────────────────────┐

└──────────────────────────────────────────────────────────────────────────────┘

  Problema                    Solución
  ───────────────────────     ────────────────────────────────────────
  Port already in use         docker-compose down -v
  MySQL unhealthy             docker-compose logs mysql
  App can't connect DB        docker-compose restart
  Image build fails           docker-compose build --no-cache
  Out of memory               Aumentar memoria en Docker Desktop
  Contenedor exited           docker-compose logs <servicio>
