└──────────────────────────────────────────────────────────────────────────────┘

  OPCIÓN 1: DESDE INTELLIJ IDEA (RECOMENDADO)

  1. Abre IntelliJ IDEA
  2. File → Open → selecciona carpeta Bolsa_Empleo
  3. Espera a que Maven cargue (2-5 minutos)
  4. Verifica JDK 17 en File → Project Structure
  5. Abre BolsaEmpleoApplication.java
  6. Haz clic en el triángulo verde ▶ junto a main()
  7. Selecciona "Run 'BolsaEmpleoApplication.main()'"
  8. Espera a ver "Started BolsaEmpleoApplication in X seconds"
  9. Abre http://localhost:8080 en el navegador

  ⏱️ Tiempo total: 5-10 minutos


  OPCIÓN 2: DESDE LÍNEA DE COMANDOS

  1. En terminal, navega a la carpeta del proyecto:
     cd c:\Users\camil\OneDrive\Documentos\PrograIV\Bolsa_Empleo

  2. Compila y ejecuta:
     mvn spring-boot:run

  3. Espera el mismo mensaje de inicio
  4. Abre http://localhost:8080

  ⏱️ Tiempo total: 3-8 minutos

┌──────────────────────────────────────────────────────────────────────────────┐
│ 💡 CONSEJOS PARA DESARROLLO                                                │
└──────────────────────────────────────────────────────────────────────────────┘

  1. USAR HOT RELOAD
     En IntelliJ: Spring Boot puede recargar cambios automáticamente
     - En Run → Edit Configurations, marca "Enable 'On Update' action"
     - Haz Ctrl+F5 para recargar sin reiniciar toda la app

  2. DEBUG CON PUNTOS DE PAUSA
     - Haz clic a la izquierda de un número de línea para agregar breakpoint
     - Ejecuta con el botón debug (insecto) en lugar del play
     - Usa F8 para seguir paso a paso

  3. VER QUERIES SQL EJECUTADAS
     En application.properties, ya está configurado:
     spring.jpa.show-sql=true
     - Ver en la consola todas las queries que ejecuta Hibernate

  4. USAR POSTMAN PARA TESTING
     - Aunque la app es HTML, puedes probar endpoints con Postman
     - Simula requests HTTP directamente

  5. LIMPIAR CACHE
     Si hay problemas raros:
     File → Invalidate Caches → Invalidate and Restart
     - IntelliJ recargará todo desde cero
