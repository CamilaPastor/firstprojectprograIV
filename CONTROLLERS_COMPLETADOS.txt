  SESSION UTIL HELPER:

  - setSessionUser(session, usuario)
  - isLoggedIn(session) → boolean
  - isTipo(session, tipoUsuario) → boolean
  - invalidateSession(session)
  - addErrorMessage(attributes, mensaje)
  - addSuccessMessage(attributes, mensaje)

  VERIFICACIÓN EN CONTROLLERS:
  private SessionUser verificarSesion(HttpSession session, RedirectAttributes attributes) {
    if (!SessionUtil.isTipo(session, "empresa")) {
      SessionUtil.addErrorMessage(attributes, "Debes iniciar sesión");
      return null;
    }
    return SessionUtil.getSessionUser(session);
  }

  LOGIN MANUAL:

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🎯 FLUJOS PRINCIPALES                                                       │
└──────────────────────────────────────────────────────────────────────────────┘

  FLUJO DE LOGIN:
  1. Usuario accede a GET /login
  2. Ve formulario con 3 opciones (empresa, oferente, admin)
  3. Completa: tipo, usuario (correo/id), password
  4. POST /login
  5. Controller valida credenciales con servicio (BCrypt)
  6. Si válido:
     - Crea SessionUser
     - session.setAttribute("usuario", sessionUser)
     - Redirige a dashboard correspondiente
  7. Si inválido:
     - Muestra error
     - Redirecciona a GET /login

  FLUJO DE REGISTRO:
  1. GET /registro/empresa o /registro/oferente
  2. Muestra formulario
  3. POST /registro/empresa o /registro/oferente
  4. Servicio registra y hashea password
  5. Redirige a /login
  6. Admin aprueba antes de poder iniciar sesión

  FLUJO: EMPRESA PUBLICA PUESTO:
  1. GET /empresa/nuevo-puesto
  2. Muestra formulario con lista de características
  3. POST /empresa/nuevo-puesto
     - idPuesto extraído de session
     - características: List<Integer>
     - niveles: List<Integer>
  4. PuestoService.publicar() guarda puesto + PuestoCaracteristica
  5. Redirige a /empresa/mis-puestos

  FLUJO: EMPRESA BUSCA CANDIDATOS:
  1. GET /empresa/buscar-candidatos?idPuesto=1
  2. BusquedaService.buscarCandidatos(idPuesto)
  3. Retorna List<ResultadoCandidato>
     - oferente: datos completos
     - cumplidos: características que cumplen
     - total: características requeridas
     - porcentaje: cumplidos/total * 100
  4. Ordenado: mayor a menor porcentaje
  5. Muestra tabla en vista
  6. Opción: GET /empresa/detalle-candidato/{id}

  FLUJO: OFERENTE SUBE CV:
  1. GET /oferente/subir-cv
  2. Muestra formulario con input file
  3. POST /oferente/subir-cv (MultipartFile archivo)
  4. Valida: debe ser PDF, max 5MB
  5. OferenteService.subirCv(idOferente, bytes, nombre)
  6. Guarda en BD (LONGBLOB)
  7. Redirige a /oferente/mi-cv

  FLUJO: DESCARGAR CV:
  1. GET /oferente/descargar-cv/{idOferente}
  2. Obtiene Cv de BD
  3. Retorna ResponseEntity<byte[]>
     - Content-Type: application/pdf
     - Content-Disposition: attachment; filename="..."
     - Body: byte[] del PDF
  4. Browser descarga automáticamente
