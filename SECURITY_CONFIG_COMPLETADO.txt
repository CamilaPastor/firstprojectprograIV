┌──────────────────────────────────────────────────────────────────────────────┐
│ 🔄 FLUJO DE AUTENTICACIÓN                                                   │
└──────────────────────────────────────────────────────────────────────────────┘

  1. Usuario accede a GET /login
  2. Completa formulario: tipo (empresa/oferente/admin), usuario, password
  3. POST /login llega a LoginController
  4. LoginController llama al servicio correspondiente:
     - EmpresaService.loginPorCorreo()
     - OferenteService.loginPorCorreo()
     - AdminService.login()
  5. Servicio valida credenciales:
     - Busca usuario en BD
     - passwordEncoder.matches(passwordSinHashear, hashedPassword)
  6. Si válido:
     - Crea SessionUser (id, nombre, correo, tipoUsuario)
     - session.setAttribute("usuario", sessionUser)
     - Redirige a dashboard correspondiente
  7. Si inválido:
     - Muestra mensaje de error
     - Redirecciona a GET /login

  8. En controllers protegidos:
     - verificarSesion() obtiene SessionUser
     - Verifica tipoUsuario
     - Si no válido: redirige con mensaje de error
