┌──────────────────────────────────────────────────────────────────────────────┐
│ 🏗️  ARQUITECTURA                                                             │
└──────────────────────────────────────────────────────────────────────────────┘

  CAPAS:
  Controller (REST)
        ↓
  Service (interfaces)
        ↓
  ServiceImpl (implementación)
        ↓
  Repository (acceso a datos)
        ↓
  Database

  FLUJO DE DATOS:
  1. Controlador recibe solicitud
  2. Llama a método del servicio
  3. Servicio valida y procesa
  4. Servicio llama a repositorio
  5. Repositorio accede a BD
  6. Datos retornan al controlador

  SEPARACIÓN DE RESPONSABILIDADES:
  ✅ Controlador: ruteo, validación HTTP
  ✅ Servicio: lógica de negocio, transacciones
  ✅ Repositorio: acceso a datos
  ✅ DTO: transferencia de datos

│ 📊 EJEMPLOS DE USO                                                          │

  REGISTRO DE EMPRESA:
  Empresa empresa = new Empresa();
  empresa.setNombre("TechCorp");
  empresa.setCorreo("info@techcorp.cr");
  empresa.setPasswordSinHashear("mipassword");
  empresa = empresaService.registrar(empresa); // Se hashea automáticamente

  LOGIN DE EMPRESA:
  Optional<Empresa> logueada = empresaService
    .loginPorCorreo("info@techcorp.cr", "mipassword");

  PUBLICAR PUESTO:
  Puesto puesto = new Puesto();
  puesto.setDescripcion("Desarrollador Java");
  puesto.setEmpresa(empresa);
  puesto.setSalario(BigDecimal.valueOf(2500000));
  puesto.setTipoPublicacion("publico");
  
  List<Integer> caracteristicas = Arrays.asList(8, 14, 11);
  List<Integer> niveles = Arrays.asList(4, 4, 3);
  
  puestoService.publicar(puesto, caracteristicas, niveles);

  BUSCAR CANDIDATOS:
  List<ResultadoCandidato> candidatos = busquedaService.buscarCandidatos(idPuesto);
  
  for (ResultadoCandidato rc : candidatos) {

    System.out.println("Porcentaje: " + rc.getPorcentaje() + "%");
  }

  AGREGAR HABILIDAD:
  oferenteService.agregarHabilidad(idOferente, idCaracteristica, 4);

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🔍 CLASE DTO: ResultadoCandidato                                            │
└──────────────────────────────────────────────────────────────────────────────┘

  UBICACIÓN:
  cr.una.bolsaempleo.dto.ResultadoCandidato

  CONTENIDO:
  - oferente: Oferente (datos del oferente)
  - cumplidos: Integer (características cumplidas)
  - total: Integer (total de características requeridas)
  - porcentaje: Double (calculado automáticamente)

  CONSTRUCTORES:
  1. Vacío (default)
  2. Con todos los parámetros
  3. Especial(Oferente, cumplidos, total) - calcula porcentaje

  FÓRMULA:
  porcentaje = (cumplidos / total) * 100
